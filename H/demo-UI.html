Demo

这套 UI 是集各家所长所诞生出来的产物。
考虑到国内有【360】和【搜狗】等众多奇葩浏览器的所谓兼容模式
这套 UI 控件兼容了 IE7，但在使用上也变得比一般 UI 框架复杂些，不过效果绝对是杠杠的。
效果和便利性不能两全，我选择的是【效果】，SO...界面调整出现难题尽管找我吧。
由于 IE7 不支持 box-sizing: border-box; 所以一旦使用了这套 UI，前端同学在调整尺寸时就要注意了

这套 UI 的 输入框和下拉框做了 大、普通、小三个 size，按钮做了大、普通、小、迷你四个 size
设计的同学请尽可能依靠这几个尺寸（特指高度，宽度调整比较简单）满足设计需求
大（高42px）、普通（高35px）、小（高28px）、迷你（高20px）
实在不行...再由前端重写尺寸吧...
理想的做法是重写尺寸将覆盖目标定在 .ui-btn 类，且尽量通过改写宽度一个属性来完成按钮大小调整

按钮的默认颜色是暗红色，UI 库在此基础上还提供了以下几种颜色的按钮：

       

假如需要自定义新颜色的按钮，可以到 public.less 文件中获取 .create-color-btn 这个 mixins 来生成按钮
例如蓝色按钮就是这样生成的：
.create-color-btn(blue, #ffffff, #4bb1cf, #4bb1cf);
生成出来的结果：
.btn-blue { background: #4bb1cf; border: 1px solid #4bb1cf; color: #ffffff; }
.btn-blue:hover { background: #53c3e4; border-color: #53c3e4; }
.btn-blue:active { background: #449fba; border-color: #449fba; }


Input

文本框的宽度会被 padding 撑大 16px，所以如果要设置 200 宽的文本框，应该是 200-16=184。随着文本框的字体大小变化，这个值还会撑大值还会改变，需要根据实际调整
如果不想兼容 IE7 直接修改 H-ui.css 为文本输入框加上 box-sizing: border-box; 样式即可
为了兼容 IE7 ~ IE8，disabled 状态需要加上 .ui-text-disabled 类

Tooltips

浮层提示，浮层进场的时候在浮层的父级增加 .z-ui-tooltips-in 类
浮层退场的时候在浮层的父级增加 .z-ui-tooltips-out 类，然后150毫秒后删除 .z-ui-tooltips-in 和 .z-ui-tooltips-out 两个类。
老规矩，兼容到 IE7，IE6 部分兼容（已经放弃它了）

箭头有12种方向，分别在父元素增加以下类别：

.ui-tooltips-top-arrow（上中） .ui-tooltips-top-left-arrow（上左）
.ui-tooltips-top-right-arrow（上右） .ui-tooltips-right-arrow（右中）
.ui-tooltips-right-top-arrow（右上） .ui-tooltips-right-bottom-arrow（右下）
.ui-tooltips-bottom-arrow（下中） .ui-tooltips-bottom-left-arrow（下左）
.ui-tooltips-bottom-right-arrow（下右） .ui-tooltips-left-arrow（左中）
.ui-tooltips-left-top-arrow（左上） .ui-tooltips-left-bottom-arrow（左下）
